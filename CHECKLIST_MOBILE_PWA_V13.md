# Checklist de Validação Mobile/PWA — VF Nexus V13

## Mobile instalado
- [ ] Instalar o app no celular pelo navegador.
- [ ] Abrir o app instalado sem aparecer tela espremida.
- [ ] Conferir se o header fica como app mobile.
- [ ] Conferir se a navegação inferior não cobre botões importantes.
- [ ] Testar login, dashboard, produtos, vendas, financeiro, relatórios, cardápio e configurações.
- [ ] Verificar se inputs não dão zoom automático.
- [ ] Verificar se modais abrem como bottom sheet.
- [ ] Verificar se tabelas longas podem ser arrastadas sem quebrar o layout.

## Versão web desktop
- [ ] Conferir se sidebar desktop continua funcionando.
- [ ] Conferir se header desktop não foi quebrado.
- [ ] Conferir se grid desktop continua largo e organizado.
- [ ] Conferir se relatórios e telas administrativas continuam funcionais.

## Paleta e logo
- [ ] Alterar cor primária.
- [ ] Alterar cor secundária.
- [ ] Alterar fundo/card/menu/texto.
- [ ] Salvar configurações.
- [ ] Fechar e abrir o app.
- [ ] Confirmar se a paleta persistiu.
- [ ] Trocar logo e confirmar no header/menu/PDF.

## Relatórios e arquivos
- [ ] Gerar PDF financeiro.
- [ ] Gerar Excel.
- [ ] Gerar ficha técnica PDF.
- [ ] Gerar orçamento de evento PDF.
- [ ] Gerar cardápio/catálogo PDF.
- [ ] Confirmar que os PDFs usam logo e paleta da empresa.
- [ ] Confirmar que não há fundo preto dominante nos PDFs.
- [ ] Confirmar que os documentos têm layout limpo, elegante e legível.

## Testes críticos
- [ ] Criar produto.
- [ ] Criar venda.
- [ ] Verificar estoque/financeiro.
- [ ] Criar cardápio.
- [ ] Editar descrição do item no cardápio.
- [ ] Gerar PDF do cardápio.
- [ ] Fechar e reabrir app para validar persistência.
